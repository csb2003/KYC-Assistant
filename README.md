# RAG-LLM Application Documentation

## Overview
This project implements a Retrieval-Augmented Generation (RAG) system using LangChain and LangGraph. It allows users to ask questions about a specific KYC (Know Your Customer) document, retrieve relevant information from the document, and receive context-aware answers generated by an LLM.

## Features
- **RAG Pipeline**: retrieves relevant document chunks to answer questions.
- **LangGraph Architecture**: orchestrates the RAG pipeline with an optional clarification loop.
- **Context-Awareness**: maintains chat history to answer follow-up questions accurately.
- **Scope Checking**: automatically detects if a question is relevant to KYC/AML and avoids answering unrelated queries.

## Requirements
- Python 3.8+
- Required libraries listed in `requirements.txt`
- Environment variables in `.env` file (see below)

## Setup
1.  **Install Dependencies**:
    ```bash
    pip install -r requirements.txt
    ```

2.  **Environment Configuration**:
    Create a `.env` file in the project root with the following variables:
    ```env
    GROQ_API_KEY="your-groq-api-key"
    HF_TOKEN="your-huggingface-token"
    ```

## Usage
### Running the Application
```bash
python agent.py
```
The application will:
1. Load and process the KYC document (`data/RBIs-Master-Direction-KYC-Direction-2016_compressed.pdf`).
2. Initialize the RAG pipeline and LangGraph agent.
3. Start an interactive chat session.

### Interactive Mode
Once the application starts, you can ask questions directly in the terminal:

```
Ask a question: What is KYC?

[LLM generates answer...]

Ask a question: [Your follow-up question]
```

To exit, type `exit`.

### API Usage (FastAPI)
For API integration, use the following endpoint:

**Endpoint**: `POST /ask`
**Body**:
```json
{
  "question": "What is the KYC process?"
}
```
**Response**:
```json
{
  "answer": "[Generated answer based on the document]"
}
```

## Code Structure
- **`agent.py`**: Main entry point containing the RAG pipeline, LangGraph agent, and interactive CLI.
- **`app.py`**: FastAPI application wrapping the RAG agent.
- **`data/`**: Directory containing the KYC PDF document.

## RAG Pipeline Components
1.  **Document Loading**: Uses `PyPDFLoader` to load the PDF document.
2.  **Text Splitting**: Splits documents into manageable chunks using `RecursiveCharacterTextSplitter`.
3.  **Embeddings**: Generates embeddings using `HuggingFaceEmbeddings` (`all-MiniLM-L6-v2`).
4.  **Vector Store**: Uses `FAISS` for efficient similarity search.
5.  **Retriever**: `create_history_aware_retriever` combines context from chat history with the current question.
6.  **LLM**: Uses `ChatGroq` (Llama 3.1 8B).
7.  **Chains**: `create_retrieval_chain` orchestrates the RAG process.

## LangGraph Agent Logic
1.  **Entry Point**: `check_scope` node.
2.  **Scope Checking**: Determines if the question is related to KYC/AML. If not, proceeds to `ask_clarification`.
3.  **RAG Execution**: If relevant, `retrieve_and_answer` node retrieves context and generates an answer.
4.  **Conditional Edges**: `clarify` function decides whether to ask for clarification or proceed with RAG based on the scope check result.
5.  **Termination**: Both branches lead to the `END` node.
